RAGAS解説

RAGAS とは？ — RAG品質評価のデファクト

RAGAS (Retrieval Augmented Generation Assessment)

LLMを活用したRAGパイプラインの品質を、自動的かつ定量的に評価するオープンソースフレームワーク。

核心アイデア: 「LLMでLLMの出力を評価する」— 人間の評価を模倣する評価用LLMが、回答の忠実性・関連性・コンテキストの質を自動スコアリング。

評価に必要な4つの入力

user_input
ユーザーの質問文
response
LLMの回答文
retrieved_contexts
RAG検索で取得した文書群
reference
正解テキスト (ground_truth)

RAGAS解説

4つの評価メトリクス — 何を測るのか

1. Faithfulness（忠実性）

回答が検索コンテキストに忠実かを評価。ハルシネーション検出。

評価プロセス:
1. 回答から主張（claims）を抽出
2. 各主張がcontextsで裏付けられるか判定
3. スコア = 裏付けあり / 全主張数
低スコアの意味: LLMがマニュアルにない情報を「でっちあげ」ている

2. Answer Relevancy（回答関連性）

回答がユーザーの質問に対して的確かを評価。

評価プロセス:
1. 回答から「元の質問」を逆生成
2. 逆生成した質問と実際の質問の類似度を計算
3. スコア = コサイン類似度の平均
低スコアの意味: 質問と無関係な情報が回答に含まれている

3. Context Precision（検索精度）

検索結果の上位に関連チャンクが来ているかを評価。

評価プロセス:
1. 各検索結果が質問に関連するかLLMが判定
2. 上位に関連チャンクが多いほど高スコア
3. スコア = Precision@K の加重平均
低スコアの意味: ChromaDB の検索ランキングが不適切

4. Context Recall（検索網羅性）

正解テキストの情報が検索結果にどれだけ含まれているかを評価。

評価プロセス:
1. ground_truth の文を分解
2. 各文が contexts に含まれるか判定
3. スコア = カバーされた文 / 全文数
低スコアの意味: 必要な情報がチャンクに含まれていない or 検索できていない

6 オイル漏れ 駐車場の地面に油のシミがある high ask_question 漏れ位置・色の確認
7 タイヤ空気圧 タイヤの空気圧警告灯がついた medium ask_question TPMS確認・空気圧点検

RAGAS評価 処理フロー

Step 1: test_cases.py から10件ロード
Step 2: 各症状で VectorStore.search()
→ contexts (検索チャンク) 取得
Step 3: RAGService.query()
→ response (LLM回答) 取得
Step 4: RAGAS SingleTurnSample に変換
user_input + response + contexts + ground_truth
Step 5: 4メトリクスで evaluate()
Faithfulness / Relevancy / Precision / Recall
Step 6: JSON出力 + コンソールテーブル表示

Usage

実行手順 — すぐに始められる

1. セットアップ

cd backend
pip install ragas datasets

.env に OPENAI_API_KEY が設定されていること（RAGASの内部評価LLMが使用）

2. RAGAS評価の実行

cd backend
python -m tests.ragas.run_ragas_eval

ChromaDB から直接検索 → LLM回答取得 → RAGAS評価。サーバー起動不要。

3. E2Eチャットテストの実行

# ターミナル1: バックエンド起動
cd backend
uvicorn app.main:app --reload

# ターミナル2: テスト実行
cd backend
python -m tests.ragas.run_e2e_chat

HTTPリクエストでチャットフロー全体を検証。サーバー起動必須。
